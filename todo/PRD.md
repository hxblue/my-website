# 产品需求文档 (PRD)：进阶版待办事项应用 (Advanced Todo App)

## 1. 项目概述
本项目是一个功能丰富的待办事项（To-Do List）单页 Web 应用。旨在提供比基础 Todo 更强大的任务管理能力，支持复杂的业务逻辑如多条件筛选、模糊搜索和数据统计，并具有极佳的用户体验和现代化的 UI 设计。

## 2. 核心功能需求
* **任务管理 (CRUD):**
    * **添加任务:** 支持设置标题（必填）、描述、截止日期、优先级（低/中/高）、分类（如工作、学习、生活）。
    * **编辑与删除:** 支持修改现有任务的所有属性，支持删除任务。
    * **状态切换:** 一键切换任务的“未完成/已完成”状态。
* **视图与检索:**
    * **多条件筛选:** 支持同时按分类（Category）、优先级（Priority）和状态（Status）进行交叉筛选。
    * **模糊搜索:** 支持在任务的标题和描述中查找关键词。
* **数据看板 (Dashboard):**
    * **统计信息:** 侧边栏实时显示总任务数、各优先级任务数、已完成数量以及整体完成率（%）。
* **数据持久化:**
    * 数据必须保存在浏览器的 `LocalStorage` 中，确保刷新页面或重新打开浏览器后数据不丢失。

## 3. 数据模型设计 (Data Model)
核心 `Todo` 对象需包含以下字段：
* `id`: string (唯一标识，如 UUID)
* `title`: string (标题)
* `description`: string (描述，可选)
* `category`: string (分类，如 'work', 'study', 'life')
* `priority`: 'low' | 'medium' | 'high' (优先级)
* `dueDate`: string | Date | null (截止日期)
* `completed`: boolean (是否完成)
* `createdAt`: string | Date (创建时间)

## 4. 扩展构思 (未来版本，本期预留接口)
* 支持多标签 (Tags)。
* 支持子任务 (Sub-tasks)。
* 支持导出数据为 CSV。
